使用Hexo和Github Actions打造自动化静态博客CI/CD Pipeline

2020-06-07

做这期视频的目的

作为程序员我们每天都会学习新技术，踩许多许多的坑。所以有一个地方能够记录和展示自己的学习技术成果就显得很重要。接下来这篇文章就是告诉大家如何打造一个简单，实用，而且免费的个人博客网站。

发布一篇博客文章的流程

发布一篇博客文章的最简单流程有以下几步。

  1. 撰写和保存博客文章
  2. 将博客文章转换成带有主题的静态网页
  3. 将静态网页部署到服务器上

以上的每一步我们都需要使用不同的工具。当撰写博客文章的时候，除了能写内容，我们还需要对不同的内容设置不同的格式。例如一级标题，二级标题，代码片段等等格式。最简单和易学的就是Markdown这种标记语言。然后我们可以上传这些Markdown文件到Github的代码仓库进行保管。

写完文章之后，我们还需要一个工具能够将我们的Markdown文章自动转换成静态网页。我推荐大家使用Hexo。它是基于Node.js打造的，能够解析Markdown语法，利用不同的主题生成静态网页内容。我们可以自己用CSS写个性化主题，也可以使用已有的开源主题。我们还可以安装各种插件，丰富博客的功能。

生成静态网页之后，我们需要将它们部署到一个服务器上，然后读者就可以通过浏览器阅读我们的文章。Github Pages提供了免费的静态网页部署服务。我们只需要将所有的静态文件上传到以<github username>.github.io的代码仓库就可以了。

最后，我们希望每次写好Markdown文件之后，都可以有个工具能够自动的调用Hexo生成金泰网页，然后上传到Github Pages使用的代码仓库里面。我们可以使用Github新推出的Github Actions定义和自动执行这些流程。

如何使用每个工具

接下来演示如何使用这些工具。Hexo需要Node.js环境。推荐使用nvm对node版本进行管理。使用Mac的用户，直接命令行运行brew install nvm。使用nvm ls-remote列出所有可使用的node版本，选择最新的稳定版本。使用nvm install v12.18.0安装最新的稳定版本。运行npm install hexo-cli -g安装Hexo命令行工具。Hexo的安装工作已经完成。

接下来演示如何使用Hexo命令行工具。运行hexo init blog初始化一个Hexo的工程，它会在当前文件夹下面创建一个叫做blog的文件夹。进入blog文件夹，运行性npm install安装所有的依赖包。运行hexo generate将样例Markdown文件转换成静态网页，然后运行hexo server启动本地服务器部署静态网页。打开浏览器，输入localhost:4000看一下初始化的博客网站。

现在我们开始写自己的博客文章。运行命令hexo new post <文章标题>，Hexo会自动生成一个以文章标题作为文件名的Markdown文件。

写完文章以后，我们准备部署到Github Pages上。登录github.com。创建一个新仓库，名字是username.github.io。username是自己Github的用户名。到刚才创建好的Hexo工程中，找到_config.yml文件，文件内容的最下面有deploy设置。加入以下配置：

github url要写上刚才新创建的username.github.io仓库的URL。branch直接用Github默认创建的master分支。我们还需要仓库的权限，允许把生成的静态网页上传到仓库里。点击Github网页右上角的头像，再点击设置，选择Developer Setting，在Personal access tokens下面生成一个新的token。在创建token的页面里面勾选repo权限。点击创建然后复制一下生成的token。在Shell里面创建一个叫做DEPLOY_BLOG的环境变量，变量值是新生成的token。再到刚才的Hexo工程中，打开_config.yml文件，在最下面deploy配置里加上token: $DEPLOY_BLOG。修改package.json中deploy script为hexo deploy -g 运行npm run deploy生成静态文件并上传过到刚才生成的代码仓库。

再到Github里面创建新的代码仓库，用来保存生成的Hexo工程文件夹。打开Github Desktop程序，点击create new repository，输入名字和描述，然后在Local Path中选择新创建的Hexo工程文件夹，点击创建。

最后我们想实现的自动化流程是，每次写完博客文章，直接push到博客工程代码仓库，然后自动生成静态网页并上传到Github Pages对应的代码仓库里。看一下如何使用Github Actions实现这个流程。打开博客工程代码仓库页面，选择Actions。点击Set up this workflow会跳转到一个YAML文件编辑页面。

简单介绍一下Github Action配置文件的语法。name是workflow的名字。on用来配置监听的Github事件，我们这里只监听master分支上的push事件。然后是配置workflow需要执行的任务，这里我们只需要一个任务就是生成部署博客静态文件。build是任务名。runs-on是运行的环境，这里选最新的ubuntu。step是这个任务的步骤。第一步使用内置的checkout action用来将代码下载到本地。第二部使用setup-node action来配置Node.js执行环境，它需要node-version这个参数来设置需要运行的node版本。最后一步部署博客。第一个先设置给git你的用户名和邮箱。

使用hexo部署的时候，需要一个token获取上传代码到仓库的权限。打开工程仓库的Github页面，点击setting，然后点击Secrets。创建新的secrets。回到workflow配置页面。运行npm install安装所有的依赖。然后运行npm run deploy。点击保存。
